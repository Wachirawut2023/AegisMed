The idea

Recreate the multidisciplinary case
conference — convened in 60 seconds

The gold-standard way hospitals actually solve hard cases is a structured review by specialists across domains. AegisMed rebuilds that as software: seven AI specialist agents instead of seven busy humans, available on demand.

1

Independent opinions

Each specialist reasons separately — real second opinions can't anchor on each other's first guess.

2

Grounded in real evidence

Citations are attached from a knowledge base, never recalled from the model's memory.

3

Token-efficient by design

Smart routing convenes only the 3–4 specialists a case actually needs, not all 7 every time.

4

Decision support, not autonomy

Every output must be verified by a licensed physician — the disclaimer is enforced throughout.

AMD platform usage — judging criterion #4

Every inference call runs on AMD

Path A — Default

Managed inference on AMD Instinct

Fireworks AI serves Google Gemma on AMD Instinct MI300X GPUs. Zero setup — every specialist and synthesis call already runs on AMD hardware the moment a Fireworks key is set.

Path B — Self-hosted

Gemma on your own AMD Developer Cloud GPU

Set LLM_BASE_URL to point AegisMed at Gemma served by vLLM/Ollama on an AMD Developer Cloud GPU instance you provision — app and model both run on AMD infrastructure, no third-party inference service in the loop.

Also on AMD Developer Cloud

The AegisMed app itself is deployed with deploy/amd-cloud.sh — a one-command script that builds and runs the Docker image on an AMD Developer Cloud instance, giving judges a live, publicly reachable demo URL. GET /health reports the active endpoint so the AMD path in use is verifiable at a glance.

Anti-hallucination

The AI is never the source of a citation

Asking an LLM to "cite sources" produces confident, plausible, and often completely fabricated references — in medicine a fabricated citation is worse than none. AegisMed follows one rule throughout the codebase: citations are always attached deterministically, never recalled from the model.

Verified references

Orphanet, OMIM, PubMed, and GARD links resolved from a ~10,700-disease knowledge base — real URLs, every time.

Guideline search links

Live, query-filtered links to PubMed, Cochrane, NICE, TRIP, MedlinePlus, NCBI Bookshelf, and Guideline Central — reordered per practice region (US/UK/EU).

Evidence-first retrieval

A retrieval agent proposes what to look up before diagnosing; the specialists cite those verified leads by name, never inventing papers or PMIDs.

AegisMed07 / 11
Evaluation approach

Scored against real, public rare-disease cases

AegisMed is evaluated on held-out cases from RareBench and CUPCase (both Apache-2.0), mixing coded symptom lists and narrative doctor write-ups — not synthetic or hand-picked examples.

The two-step harness

  • python data/build_dataset.py — downloads & converts public cases (free, no key)
  • python eval/run_eval.py — runs the full board on each case, checks whether the known diagnosis (or a synonym) appears in the output
  • Mirrors the real app flow: intake questions are auto-answered from the case, then the board runs

The headline metric

"The correct diagnosis was surfaced in X% of held-out cases" — written to eval/results.md, broken down per data source.

Requires a real FIREWORKS_API_KEY to be meaningful — demo mode returns fixed sample output, so the harness warns explicitly when scores wouldn't be real.

Market & roadmap

From solo clinician to regulated deployment

  • Solo clinicians — free second opinions on hard cases
  • Medical schools — teaching mode grades vs. board
  • Hospitals — case history & team collaboration (B2B)
